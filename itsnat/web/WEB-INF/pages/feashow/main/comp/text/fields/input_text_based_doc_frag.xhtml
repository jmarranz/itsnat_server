HTML <input type="text">, <input type="password"> and <input type="file"> elements can be attached to ItsNatHTMLInputText, ItsNatHTMLInputPassword and ItsNatHTMLInputFile components respectively.

Any change to the text field in the client is notified to the server accordingly, the property "value" in server is updated and the component data model is updated and notified. If the component data model in server is updated the client is updated too.

The calls ItsNatHTMLInput.focus() and ItsNatHTMLInput.select() used in this example are the remote versions of the W3C DOM HTMLInputElement focus and select methods (in Batik DOM these methods are dummy) they send to the client the input element focus() and select() calls.

In UCWEB, text controls do not fire events, when the client contacts the UCWEB server any change in text controls is automatically transmitted. Bolt has a similar problem.
